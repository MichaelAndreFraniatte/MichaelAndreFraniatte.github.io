Account File Authentication for Open Program

I've made a method to authenticate a user with a file for allowing to open a program or not. It need to authenticate the user and the computer used. Let's see...

Considerations About the Program

It's made for secure a program, and there is a control in real time for account maintenance if the user redeem the program. If the user needs to use the program on another computer, it need a support.

Help and Tips

The process use to secure a program is made with LocalAuthentication and FileAuthentication to control who is authorized to open. You must create files of the checksum of assemblies, like exe and DLL. Then, it need to ask the user a key corresponding to his PC unique identifier (ID) but crypted in a file he generate, you decrypt after he sends you the file. The user execute an exe for generate his key. After having this key decrypted, it need to create the key file corresponding to a file containing ID recrypted. Finally, the ID decrypted in Authorized file, is linked with email of the user and a boolean to set if the user is authorized or unauthorized to open, in case he redeemed. And also, the ID is used to check if the user can open in relation to his PC. The process is as following.

  • GetChecksum (exe, dll) -> Assembly -> secure from modification.
  • GetKey (.me) -> GetKeyDecryptor (.txt) -> get the key.
  • KeyFileCreator (.txt) -> Assembly (.kf) -> add the key.
  • LocalAuthentication -> PC -> check the key.
  • FileAuth (repo) -> Authorized (edit) -> check the authorization.
